## Python и venv (урок 2.2)

### Зачем
Python — язык, на котором сервер говорит с базой. Виртуальное окружение `.venv` — **отдельная чистая полка** только для инструментов одного проекта, чтобы они не смешались с другими.

### Как
Перейди **в папку своего серверного проекта** и запусти:
```bash
cd ~/путь/к/твоему/серверному/проекту
bash /путь/к/course-setup-runbook/99-appendix-backend/install-python-venv.sh
```
Скрипт спросит подтверждение, создаст `.venv`, поставит три пакета и сделает `requirements.txt`.

**Чек-поинт:** в строке Терминала появляется `(.venv)`, а в `requirements.txt` есть `fastapi`, `pydantic-settings`, `asyncpg`.

> 🚫 Не коммить `.venv` в GitHub (тяжёлая папка, её каждый компьютер собирает сам). А вот `requirements.txt` — коммить.

## Деплой на Railway (урок 2.7) — это веб, не Терминал

Railway в курсе делается **через сайт**, поэтому «волшебной кнопки»-скрипта тут нет — это была бы выдумка. Порядок (всё в браузере + одна строка в настройках сервиса):

1. Зафиксируй список пакетов: `pip freeze > requirements.txt` и закоммить его в GitHub.
2. На **railway.com** → `New Project` → `GitHub repo` → выбери репозиторий.
3. В сервисе → `Variables` → добавь `DATABASE_URL`.
4. В сервисе → `Settings` → **Start Command**:
   ```text
   uvicorn main:app --host 0.0.0.0 --port $PORT
   ```
   (сервер обязан слушать порт из переменной `$PORT`, которую выдаёт Railway).
5. В сервисе → `Settings` → `Networking → Public Networking` → **Generate Domain**.

**Чек-поинт:** открой `https://твой-домен.railway.app/docs` — если видишь меню FastAPI, сервер живёт в интернете.

> 🚫 Не нажимай необратимое без явного «да»: удалить сервис, удалить базу, отключить проект. Не покупай домен — хватает домена Railway.
