# 06w · Первый сайт — публикуем в настоящий интернет (веб-трек)

> ⭐ Фаза веб-трека, требует пройденной фазы **05w** (Netlify). Мобильный трек (05/06) для неё
> не нужен.

## Зачем это
Первая победа веб-трека — **твой сайт по настоящей ссылке в интернете**, которую можно открыть
с телефона и отправить другу. По дороге собираем главный ритм веб-разработки — тот же, что
у профессионалов:

```text
правка → commit (Git запомнил) → push (GitHub сохранил) → deploy (Netlify опубликовал)
```

## Шаги по порядку

### Шаг 1. Создать сайт и увидеть его
**Что произойдёт:** появится папка `~/vibecoding/vibecoding_first_site` с готовой страницей
(заголовок + кнопка-счётчик), в проект ляжет **AGENTS.md** — правила для ИИ-агента (коммит перед
деплоем, публикация только с твоего «да»), и сайт откроется в браузере — пока только у тебя.
```bash
bash 06w-first-site/create-site.sh
```
**🎉 Видимая победа №1:** страница на экране, кнопка **`+`** считает нажатия.

### Шаг 2. Первая правка руками ИИ
**Что произойдёт:** ты попросишь ИИ изменить один текст в одном файле. В Терминале:
```bash
cd ~/vibecoding/vibecoding_first_site
codex
```
(или `claude` — кто тебе больше нравится). Вставь точный промпт:
```text
Ты работаешь в моём учебном сайте. Прочитай AGENTS.md и следуй ему. Сделай одну маленькую правку: в файле index.html измени заголовок на "Мой первый сайт в интернете". Не меняй кнопку и счётчик. Не трогай другие файлы. После изменения кратко перечисли, что поменял. Ничего не публикуй — публиковать будем отдельным шагом.
```
Обнови страницу в браузере (⌘R) — заголовок изменился.

### Шаг 3. Сохранить и опубликовать
**Что произойдёт:** скрипт покажет, что изменилось, сделает коммит, с твоего согласия отправит
проект в приватный репозиторий GitHub и — снова с твоего согласия — опубликует сайт на Netlify.
```bash
bash 06w-first-site/publish-site.sh
```
**🎉 ГЛАВНАЯ ПОБЕДА:** в выводе появится ссылка (Website URL). Открой её на телефоне —
это твой сайт в настоящем интернете.

### Шаг 4. Проверка
```bash
bash 06w-first-site/verify.sh
```

## Как обновлять сайт дальше (запомни этот ритм)
Любое изменение: попроси ИИ сделать правку → посмотри глазами → снова
`bash 06w-first-site/publish-site.sh`. Он закоммитит, запушит и переопубликует.
Именно так ты будешь вести все свои будущие сайты.

> 💡 **Бонус (по желанию): авто-деплой из GitHub.** Можно привязать репозиторий к Netlify так,
> чтобы КАЖДЫЙ push в main публиковался сам, без команды deploy: в папке сайта запусти
> `netlify init` → «Connect this directory to an existing project» не нужно, выбирай
> создание/привязку к репо и оставляй build-настройки пустыми (сайт статический). Это тот же
> механизм, что у больших команд: main = то, что видит интернет.

## Чек-поинт фазы
Фаза пройдена, если:
- сайт лежит в `~/vibecoding/vibecoding_first_site` и в нём есть `AGENTS.md`;
- есть коммит и репозиторий на GitHub;
- сайт открывается по ссылке Netlify (глазами, лучше с телефона).
