# 05w · Netlify — издательство сайтов (веб-трек)

> ⭐ **Эта фаза — по желанию** (веб-трек). Нужна тем, кто хочет делать **сайты и веб-платформы**
> и сразу публиковать их в интернете. Мобильный трек (Flutter, фаза 05) для этого не нужен —
> треки независимы: можно пройти один, другой или оба.

## Зачем это
**Netlify** — это издательство сайтов: ты отдаёшь ему папку с сайтом, а оно публикует её в
интернете и даёт настоящую ссылку, которую можно открыть с любого телефона и отправить друзьям.
Бесплатного тарифа для учёбы более чем достаточно. **Netlify CLI** — пульт к издательству
прямо из Терминала.

Схема, которую мы собираем (главный ритм веб-разработки):

```text
правка → commit (Git запомнил) → push (GitHub сохранил) → deploy (Netlify опубликовал)
```

## Шаги по порядку

### Шаг 1. Поставить CLI и войти в Netlify
**Что произойдёт:** установится netlify-cli (через npm из фазы 04), затем откроется браузер —
там нужно войти в аккаунт Netlify (или сначала завести его: бесплатно, удобнее всего кнопкой
**Sign up with GitHub**) и нажать «Authorize».
```bash
bash 05w-netlify/install-netlify.sh
```

### Шаг 2. Проверить
```bash
bash 05w-netlify/verify.sh
```
**Что ты увидишь:** версию CLI и подтверждение входа.

## Чек-поинт фазы
Фаза пройдена, если:
- `netlify --version` показывает версию;
- `netlify status` показывает твой email (вход выполнен).
