# TROUBLESHOOTING — типичные ошибки и что делать

Здесь — частые затыки и простые решения. Правило для Codex: сначала загляни сюда, объясни новичку **простыми словами**, дай **один** следующий шаг. Почти ничего из этого не значит «сломал Mac».

---

## `command not found: brew`
Homebrew установлен, но Терминал не знает, где он лежит (не настроен PATH).
1. Закрой Терминал **полностью** и открой заново.
2. Проверь: `brew --version`.
3. Если всё ещё нет — выполни строку для своего чипа:
   - Apple Silicon: `eval "$(/opt/homebrew/bin/brew shellenv)"`
   - Intel: `eval "$(/usr/local/bin/brew shellenv)"`
4. Повтори `bash 02-foundation/install-homebrew.sh` — он допишет PATH в `~/.zprofile` без дублей.

## Mac просит пароль, но «ничего не печатается»
Это нормально. Пароль в Терминале вводится **невидимыми чернилами** — символы не отображаются. Просто набери пароль от Mac и нажми Enter.

## Установка Homebrew зависла на длинном тексте
Homebrew подробно пишет, что делает. Дождись **новой строки с мигающим курсором** — значит закончил. Не закрывай окно раньше времени.

## `xcode-select: note: install requested for command line tools`
Появилось окно Apple «Установить инструменты командной строки» — нажми **Установить**, дождись конца, потом запусти скрипт заново.

## `gh auth login` не открыл браузер / вход не завершился
1. Повтори вручную: `gh auth login --web`.
2. Выбирай: **GitHub.com → HTTPS → вход через браузер**.
3. Проверь: `gh auth status`.
Вход бывает капризным с первого раза — повторный запуск обычно решает.

## Не вижу окна Терминала или браузера (агент запустил их «у себя»)
Если помощник (Codex/Claude/GPT) выполняет команды или открывает вход **внутри своего окна**, ты
этого не увидишь — и вход/пароль завершить не сможешь. Всё должно идти в **твоём** настоящем
Терминале и **твоём** браузере.
1. Скажи помощнику прямо: «Я не вижу окна. Дай мне команду/ссылку — я сам открою в своём Терминале и браузере».
2. Команды вставляй в **штатный Терминал** (Terminal.app), вход по ссылке открывай в **своём браузере** (Safari/Chrome), где ты уже залогинен.
3. Пароль Mac и окна установки (App Store, Apple) появляются в самой системе — вводи там.
Правило: если чего-то «не видно на экране» — значит шаг ушёл в окно помощника; переведи его в своё.

## Нет аккаунта GitHub
Заведи бесплатно: **https://github.com/signup**. Потом вернись к `03-git-github/install-gh.sh`.

## `claude` или `codex` вместо версии открыли вход в аккаунт
Это **хороший знак** — инструмент установлен и ждёт задания. Войди (Claude — через браузер, Codex — через ChatGPT) и продолжай.

## `npm i -g …` ругается на права (EACCES / permission denied)
Если Node ставился через Homebrew, прав хватает. Если права всё же мешают:
1. Проверь, что Node из Homebrew: `which node` → должен быть в `/opt/homebrew` (Apple Silicon) или `/usr/local` (Intel).
2. **Не используй `sudo npm`** — это создаёт новые проблемы. Лучше переустанови Node из Homebrew: `brew reinstall node`.

## `flutter doctor` показывает красные крестики
Это **не поломка**, а список дел. Частые пункты:
- **cmdline-tools / Android licenses** → `flutter doctor --android-licenses` (отвечай `y`).
- **Xcode** → доустанови из App Store и запусти `05-flutter/install-xcode-tools.sh`.
- **CocoaPods** → `05-flutter/install-cocoapods.sh`.
Скопируй **весь** вывод `flutter doctor` и отдай Claude Code с просьбой объяснить ОДИН следующий шаг.

## `flutter: command not found` после установки
Flutter ставился каском, но PATH не подхватился в этой сессии. Закрой Терминал и открой заново. Если не помогло — проверь `ls /opt/homebrew/bin/flutter` (Apple Silicon) и сообщи Claude Code.

## `flutter run` не видит устройства / «No devices found»
1. Открой симулятор: `open -a Simulator` и подожди, пока он полностью загрузится.
2. Проверь: `flutter devices` — в списке должен быть iPhone Simulator.
3. Запусти снова `flutter run` и при вопросе выбери номер iPhone Simulator.

## Первая сборка `flutter run` идёт очень долго
Нормально для первого раза — Flutter «разогревает станок». Не закрывай окно. Жди строку, где приложение открылось на симуляторе.

## Как остановить `flutter run`
Нажми **`q`** в окне Терминала, где он работает. Для горячей перезагрузки после правки — **`r`**.

## `git commit` говорит «nothing to commit»
Изменений нет. Сначала попроси Claude Code/Codex сделать маленькую правку текста в `lib/main.dart` (см. `06-first-win/runbook.md`, Шаг 2), потом запусти `first-edit-commit.sh` снова.

## `gh repo create` — «name already exists»
Репозиторий с таким именем уже есть. Это не страшно: проект, скорее всего, уже привязан — просто отправь изменения `git push -u origin main`, либо создай с другим именем.

## ИИ изменил слишком много файлов
Остановись. Попроси помощника: «Откати лишние изменения и оставь только правку текста в lib/main.dart». Проверь `git status` — добавляй в коммит только нужный файл.

## Я случайно увидел `.env` или секрет в изменениях
**Не коммить.** Секреты в GitHub не отправляем. Добавь `.env` в `.gitignore`, убери из индекса `git rm --cached .env`. `first-edit-commit.sh` тоже остановится, если заметит `.env`.

## Коммит не проходит: «похоже, в коммит попал секрет»
Сработал сторож секретов (gitleaks) — он нашёл в изменениях что-то похожее на ключ, токен или пароль и остановил коммит. Это защита, а не поломка.
1. Открой файл, который добавляешь, и найди строку с ключом/паролем/токеном.
2. Убери секрет из файла. Если он нужен приложению — положи его в файл `.env` (он уже игнорируется и в GitHub не уходит).
3. Если добавлял файл с секретом — убери его из индекса: `git rm --cached <файл>`.
4. Повтори коммит.
Если уверен, что это **не** секрет (ложная тревога) — скопируй сообщение и покажи Claude Code, разберитесь вместе, прежде чем обходить защиту.

## В проекте с husky (или своим core.hooksPath) сторож секретов молчит
Некоторые проекты сами задают `core.hooksPath` — например, husky прописывает её в локальный
конфиг репозитория. Локальная настройка Git сильнее глобальной, поэтому в таких проектах
запускаются только хуки самого проекта, а глобальный сторож секретов — нет. Это поведение Git,
а не поломка. Нужен сторож и там — добавь вызов gitleaks в pre-commit самого проекта
(попроси Claude Code). Обратное неверно: наш глобальный хук локальные `.git/hooks/pre-commit`
не отключает — он сам передаёт им управление в конце.

## Настройки Mac не применились (скрытые файлы, Dock)
Иногда `defaults` подхватываются только после перезапуска служб или перезагрузки.
1. Запусти `bash 01-macbook-setup/setup-mac-defaults.sh` ещё раз — это безопасно.
2. Или вручную: `killall Finder && killall Dock`.
3. Если всё ещё нет — перезагрузи Mac.
Проверить скрытые файлы: `defaults read com.apple.finder AppleShowAllFiles` (должно быть `1`).

## Хочу откатить настройки Finder назад
```bash
defaults delete com.apple.finder AppleShowAllFiles
defaults delete com.apple.finder _FXShowPosixPathInTitle
killall Finder
```

## Скрипт упал на середине — что делать
Все скрипты идемпотентны: **запусти тот же скрипт ещё раз**. Он пропустит уже сделанное и доделает остальное. Если падает в одном и том же месте дважды — скопируй ошибку Claude Code/Codex и разберите вместе.

---

> Если ничего не помогло за 2–3 попытки — это не провал. Остановись, передохни, и попроси ИИ-помощника объяснить последнюю ошибку простыми словами и дать один безопасный шаг.
