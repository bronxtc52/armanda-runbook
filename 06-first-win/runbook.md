# 05 · Первая победа — запускаем настоящее приложение

## Зачем это
Сегодня будет **первая настоящая победа**. Мы не пишем приложение с нуля — Flutter сам создаст готовый учебный проект (как набор LEGO с инструкцией), а мы соберём и запустим его на игрушечном iPhone. Потом сделаем первую маленькую правку руками ИИ, увидим её на экране и сохраним — это и есть главный ритм курса: **изменил → увидел → сохранил**.

## Шаги по порядку

### Шаг 1. Создать и запустить приложение
**Что произойдёт:** появится папка `~/vibecoding/vibecoding_first_app`, откроется симулятор iPhone, и приложение-счётчик запустится. Первая сборка идёт долго — не закрывай окно.
```bash
bash 06-first-win/create-and-run.sh
```
**🎉 Видимая победа:** на экране игрушечного iPhone появилось приложение, и кнопка **`+`** увеличивает число. Это не макет и не картинка — настоящее приложение.

> Чтобы остановить приложение — нажми **`q`** в окне Терминала, где идёт `flutter run`. Хочешь оставить запущенным — пока не нажимай.

### Шаг 2. Первая правка руками ИИ
**Что произойдёт:** ты попросишь Claude Code изменить один текст в одном файле. Открой **новое окно Терминала** и:
```bash
cd ~/vibecoding/vibecoding_first_app
claude
```
Вставь точный промпт (одна задача — один файл):
```text
Ты работаешь в моём учебном Flutter-проекте. Сделай одну маленькую правку: в файле lib/main.dart измени видимый текст так, чтобы на экране было понятно: "Моё первое приложение". Не меняй логику счётчика. Не трогай другие файлы. После изменения кратко перечисли, какие строки по смыслу изменил.
```
Потом вернись в окно с `flutter run` и нажми **`r`** (hot reload — как заменить табличку на двери, не перестраивая дом). На симуляторе должна появиться **новая надпись**.

### Шаг 3. Сохранить правку и отправить в GitHub
**Что произойдёт:** скрипт покажет, что изменилось, сделает аккуратный коммит и отправит проект в твой приватный репозиторий на GitHub.
```bash
bash 06-first-win/first-edit-commit.sh
```
**Увидишь:** `✅ Коммит создан` и ссылку/подтверждение, что проект в облачном сейфе.

### Шаг 4. Проверить
```bash
bash 06-first-win/verify.sh
```

## Чек-поинт фазы
Фаза пройдена, если:
- приложение **запустилось** на симуляторе и кнопка `+` работает (это видно глазами);
- после правки ИИ на экране появился **новый текст**;
- сделан **коммит** и проект **отправлен в GitHub**.
