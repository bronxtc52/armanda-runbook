# 03 · Git и GitHub

## Зачем это
Появятся два инструмента. **Git** запоминает версии твоего проекта: в любой момент можно
сделать снимок, а если что-то сломается — вернуться к рабочему снимку назад. **GitHub** хранит
копию проекта в интернете — она не пропадёт, даже если что-то случится с ноутбуком. ИИ-помощникам
с GitHub тоже удобнее: им проще понимать, что менялось.

## Шаги по порядку

### Шаг 1. Поставить Git
**Что произойдёт:** через Homebrew поставится Git.
```bash
bash 03-git-github/install-git.sh
```
**Увидишь:** `✅ Сработало: git version 2.x.x`.

### Шаг 2. Поставить GitHub CLI и войти в GitHub
**Что произойдёт:** поставится `gh`, потом откроется **браузер для входа в GitHub**. Скрипт остановится и подождёт, пока ты войдёшь сам. В вопросах выбирай: **GitHub.com → HTTPS → вход через браузер → подтвердить**.

> 🔑 Нужен аккаунт GitHub. Если его нет — заведи бесплатно на **https://github.com/signup**, потом запусти скрипт.

```bash
bash 03-git-github/install-gh.sh
```
**Увидишь:** `✅ Ты вошёл в GitHub`.

### Шаг 3. Подписать свои снимки именем и email
**Что произойдёт:** скрипт спросит твоё имя и email и пропишет их как «подпись» на снимках. Лучше указать тот же email, что на GitHub.
```bash
bash 03-git-github/setup-git-identity.sh
```
> Можно и без вопросов — передать заранее:
> ```bash
> GIT_AUTHOR_NAME="Имя Фамилия" GIT_AUTHOR_EMAIL="you@example.com" bash 03-git-github/setup-git-identity.sh
> ```

### Шаг 4. Включить защиту от утечки секретов
**Что произойдёт:** поставится небольшой сторож (gitleaks), который проверяет каждый коммит и не
даёт случайно отправить в GitHub ключ, токен или пароль. Секреты в GitHub попадать не должны:
репозиторий видят другие, и утёкший ключ пришлось бы срочно менять. Настройка общая — работает
во всех твоих будущих проектах, ничего не надо делать заново.
```bash
bash 03-git-github/setup-secret-guard.sh
```
**Увидишь:** `✅ Сторож секретов включён`. С этого момента проверка идёт сама перед каждым коммитом.

> 🔑 Правило простое: ключи и пароли **никогда** не пишем прямо в файлы проекта. Если приложению
> нужен ключ — кладём его в файл `.env` (он уже игнорируется и в GitHub не уходит).

### Шаг 5. Проверить всё разом
```bash
bash 03-git-github/verify.sh
```

## Чек-поинт фазы
Фаза пройдена, если:
- `git --version` и `gh --version` показывают версии;
- `gh auth status` пишет, что ты вошёл в GitHub;
- подпись Git (имя + email) задана;
- сторож секретов включён (gitleaks установлен, проверка перед коммитом работает);
- `verify.sh` закончился «Фаза 03 пройдена».
