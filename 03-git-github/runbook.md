# 02 · Git и GitHub — машина времени и облачный сейф

## Зачем это
Появятся две суперспособности. **Git** — машина времени: в любой момент делаешь снимок проекта, и если что-то сломается, есть точка возврата. **GitHub** — облачный сейф: копия проекта в интернете, она не пропадёт, даже если с ноутбуком что-то случится. ИИ-помощники тоже любят GitHub — им проще понимать, что изменилось.

## Шаги по порядку

### Шаг 1. Поставить Git
**Что произойдёт:** через Homebrew поставится Git.
```bash
bash 03-git-github/install-git.sh
```
**Увидишь:** `✅ Сработало: git version 2.x.x`.

### Шаг 2. Поставить GitHub CLI и войти в GitHub
**Что произойдёт:** поставится `gh`, потом откроется **браузер для входа в GitHub**. Скрипт остановится и подождёт, пока ты войдёшь сам. В вопросах выбирай: **GitHub.com → HTTPS → вход через браузер → подтвердить**.

> 🔑 Нужен аккаунт GitHub. Если его нет — заведи бесплатно на **https://github.com/signup**, потом запусти скрипт.

```bash
bash 03-git-github/install-gh.sh
```
**Увидишь:** `✅ Облачный сейф открыт — ты вошёл в GitHub`.

### Шаг 3. Подписать свои снимки именем и email
**Что произойдёт:** скрипт спросит твоё имя и email и пропишет их как «подпись» на снимках. Лучше указать тот же email, что на GitHub.
```bash
bash 03-git-github/setup-git-identity.sh
```
> Можно и без вопросов — передать заранее:
> ```bash
> GIT_AUTHOR_NAME="Имя Фамилия" GIT_AUTHOR_EMAIL="you@example.com" bash 03-git-github/setup-git-identity.sh
> ```

### Шаг 4. Проверить всё разом
```bash
bash 03-git-github/verify.sh
```

## Чек-поинт фазы
Фаза пройдена, если:
- `git --version` и `gh --version` показывают версии;
- `gh auth status` пишет, что ты вошёл в GitHub;
- подпись Git (имя + email) задана;
- `verify.sh` закончился «Фаза 03 пройдена».
